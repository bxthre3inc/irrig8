**⚠️ NEW: AOS-ASYNC-001 — Race condition in mesh-sync.test.ts:116**

The test `"state sync propagates across chain"` in `tests/mesh-sync.test.ts:111-116` fails intermittently:

```
error: expect(received).toBe(expected)
Expected: 42
Received: undefined

at <anonymous> (/home/workspace/Bxthre3/projects/agentos/tests/mesh-sync.test.ts:116:54)
```

**Root cause:** `setTimeout` callback fires before state propagates — synchronous broadcast doesn't wait for async propagation.

```typescript
mesh.propagateState("node-A", "counter", 42);
setTimeout(() => {
  expect(nodeB.state.getState("counter")?.value).toBe(42); // nodeB.state is undefined
}, 0);
```

**Impact:** This is a **test flakiness issue**, not a code defect. The underlying mesh sync logic likely works correctly — the test doesn't await properly.

| ID | Description | Severity | Status |
|----|-------------|----------|--------|
| AOS-ASYNC-001 | mesh-sync.test.ts:116 race condition — test fails to await propagation | P2 | 🟡 OPEN — Mesh-Engineer |

**Recommended fix:** Replace `setTimeout` with `await` using Mesh event subscription or a mock clock.
