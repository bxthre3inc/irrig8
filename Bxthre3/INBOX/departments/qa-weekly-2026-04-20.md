### VPC-004 Analysis (Root Cause Identified)

`system_configs` table IS defined in `db.ts` → `initDatabase()`. The table is created correctly with:
```sql
CREATE TABLE IF NOT EXISTS system_configs (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  description TEXT,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

**Root cause:** `unified_engine.test.ts` calls `processWager()` from `engine/core.ts` without calling `initDatabase()` first. The test file does not initialize the DB before running game engine tests. This means `system_configs` table is never created in the test DB context, causing the "no such table" failure.

**Fix:** Add `await initDatabase()` to the top of `unified_engine.test.ts` before any engine tests run. Alternatively, add DB initialization to a `beforeAll` hook.
