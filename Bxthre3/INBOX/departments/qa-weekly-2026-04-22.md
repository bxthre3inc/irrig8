## 🔴 RESOLVED: VPC-004 — Unified Engine Tests Fixed ✅

**Root cause:** Two issues were blocking `unified_engine.test.ts`:

1. **Missing `initDatabase()`** — The test file had no DB initialization, so `system_configs` table didn't exist in the test DB context. This caused RiskManagementService to fail when checking house liquidity.

2. **Missing wallet + RNG seed** — `rng_seeds` table has `FOREIGN KEY (user_id) REFERENCES wallets(user_id)`. Since no wallet was seeded for `test-player-123`, the FK constraint failed when `RNGPersistenceService` tried to INSERT a new RNG seed.

**Fix applied (2026-04-22):**

`unified_engine.test.ts` now includes:
- `import { initDatabase, db } from './db'` and `import { randomBytes, createHash } from 'crypto'`
- `beforeAll()` hook calling `await initDatabase()`
- Wallet seed: `INSERT OR IGNORE INTO wallets (user_id, balance) VALUES (?, 10000)`
- RNG seed: `INSERT OR IGNORE INTO rng_seeds (user_id, server_seed, client_seed, hashed_server_seed, nonce)` with proper SHA256 hash
- **Wager reduced from 100 → 10 VLY** — 100 VLY exceeded the default house liquidity ceiling of ~15.5 VLY (house_bankroll_reserve = 100.00, max_payout_cap = 80.00 → max safe wager ≈ 15.5 VLY)

**Test results after fix:**
```
✓ Unified Core Engine Expansion > should process a Coin Toss wager through the core pipeline [5.50ms]
✓ Unified Core Engine Expansion > should process a Blackjack wager through the core pipeline [6.73ms]
```
