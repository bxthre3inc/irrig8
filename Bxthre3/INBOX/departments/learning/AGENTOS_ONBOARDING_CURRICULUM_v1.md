**Version:** 1.0

## Program Overview

The AgentOS Onboarding Program is a 30-day structured curriculum for every agent joining the Bxthre3/AgentOS workforce. It covers foundational knowledge, role-specific tools, cross-agent collaboration, and measurable skill milestones.

**Goal:** Every agent fully operational and self-sufficient within 30 days.

---

## Prerequisites

Before Day 1, the following must be completed by the onboarding manager (typically the department lead):
- [ ] Agent created in `org.ts` with correct role, department, managerId, skills, tools
- [ ] Agent INBOX file created at `Bxthre3/INBOX/agents/{agent-id}.md`
- [ ] Agent added to appropriate department INBOX
- [ ] Tool access provisioned per agent's `tools` list in org.ts

## Day 1 — Foundation

### Morning (9:00 AM - 12:00 PM)

| Topic | Duration | Format | Owner |
|---|---|---|---|
| Welcome to Bxthre3 — Mission, Vision, Ventures | 30 min | Presentation | brodiblanco or Zoe |
| SOUL.md — Behavioral identity and principles | 45 min | Reading + Discussion | Learn |
| AGENTS.md — Workspace memory and routing | 45 min | Reading + Lab | Learn |

**Lab 1:** Navigate the workspace. Find and open:
1. Your agent INBOX
2. Your department INBOX
3. The canonical INBOX
4. SOUL.md and AGENTS.md

### Afternoon (1:00 PM - 5:00 PM)

| Topic | Duration | Format | Owner |
|---|---|---|---|
| Canonical names — product names that matter | 30 min | Briefing | Learn |
| INBOX Routing System — how information flows | 60 min | Lab | Learn |
| Agent roster — who does what (org.ts walkthrough) | 30 min | Interactive | Learn |

**Lab 2:** INBOX routing practice:
1. Write a mock report to your agent INBOX
2. Write a department update to your department INBOX
3. Identify what goes to the canonical INBOX (P0/P1 only)

**Deliverable:** Complete and submit the Day 1 Reflection form.

## Day 7 — Week 1 Check-in

**Format:** 30-min 1:1 with department lead

| Topic | Discussion Points |
|---|---|
| Clarity of role | Does the agent understand their responsibilities? |
| Skill gaps identified | What training is still needed? |
| Tool proficiency | Which tools does the agent need help with? |
| blockers | What's preventing the agent from executing? |

**Deliverable:** Week 1 Skills Self-Assessment submitted to Pulse.

---

## Day 14 — Week 2 Review

**Format:** 45-min review with department lead + Learn

| Topic | Milestone |
|---|---|
| First task completion | Was the task done well? On time? |
| INBOX proficiency | Can the agent route correctly without help? |
| Skill gaps closed | Which training needs were addressed? |
| New skill gaps identified | What additional training is needed? |

**Deliverable:** Updated skills profile in agent INBOX.

---

## Day 30 — Month 1 Review

**Format:** 60-min review with department lead + Learn + Pulse

| Topic | Evaluation |
|---|---|
| Role readiness | Can the agent operate independently? |
| Certification plan | What certifications will the agent pursue? |
| Next 90-day goals | What does the agent need to master? |
| Cross-department skills | Which cross-functional skills does the agent need? |

**Deliverable:** 
- Updated skills matrix entry in `Bxthre3/AGENTS_SKILLS_MATRIX.md`
- Certification plan document in agent INBOX

## Role-Specific Onboarding Supplements

### Engineering Agents
**Additional Days 3-5:**
- Code review standards (per SOUL.md Principle #4: Ship With Integrity)
- Git workflow: commit early, commit often, no broken main
- Test infrastructure: bun test for new features
- Service monitoring: Loki logs, Sentinel health checks

### Field Operations (Irrig8, RAIN)
**Additional Days 3-5:**
- Irrig8 product deep-dive: satellite + sensor data → deterministic watering
- SLV-specific context: San Luis Valley conditions, soil types
- Sensor suite hardware overview
- Water law fundamentals (RAIN agent)

### Gaming Operations (VPC)
**Additional Days 3-5:**
- Sweepstakes gaming compliance framework
- KYC/AML protocols
- Cash payment handling procedures
- VPC platform walkthrough
