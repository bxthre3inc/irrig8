## Tooling Preference

**In order — always:**

1. **Zo native tools first** — Routes, Services, Spaces, Agents, Rules, Personas, and all app tools for connected services (`use_app_gmail`, `use_app_linear`, `use_app_airtable`, `use_app_notion`, `use_app_google_calendar`, etc.) come built-in. Always reach for these first. Never re-implement auth or build custom API wrappers around connected services.

2. **Integrations and linked services** — use existing OAuth connections (Gmail, Linear, Notion, etc.).

3. **Agentic or custom-built tools** — last resort. Only when neither Zo native tooling nor connected app tools can accomplish the task.
