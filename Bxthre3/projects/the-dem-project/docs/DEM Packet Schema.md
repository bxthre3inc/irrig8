# Technical Specification: DEM Packet Schema (Worksheet v1.0)

## 1. Overview

The **Deterministic Execution Manifest (DEM)**, internally referred to
as the **Worksheet**, is a JSON-serialized logic package generated by
the **Inference Engine (Zo)** and transmitted to the **Edge Computing
Gateway (Hub)**. This schema ensures that the Hub has all necessary
\"Cognitive Utility\" to perform autonomous actuations without cloud
pings.

## 2. Header Block (Metadata & Security)

Every DEM packet begins with a metadata header to ensure authenticity
and temporal relevance.

{\
\"manifest_id\": \"DEM-2026-06-29-SLV-001\",\
\"gateway_id\": \"HUB-7742-MV\",\
\"timestamp_created\": \"2026-06-28T22:00:00Z\",\
\"valid_from\": \"2026-06-29T00:00:00Z\",\
\"valid_until\": \"2026-06-29T23:59:59Z\",\
\"firmware_req\": \"v2.4.1\",\
\"auth_hash\":
\"sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855\"\
}

## 3. Environmental Context (The Virtual Grid)

This section contains the distilled outputs from the **Spatial Query
Engine (SQE)** and **Kriging** models, providing the Hub with a
localized \"Environmental Map.\"

\"context\": {\
\"field_id\": \"SECTION-12-NORTH\",\
\"soil_variability_index\": 0.84,\
\"kriging_weights\": \[0.12, 0.45, 0.22, 0.21\],\
\"predicted_et_rate\": 0.28,\
\"baseline_moisture_target\": 32.5\
}

## 4. Decision Logic (The Asynchronous Tree)

This is the core \"Worksheet.\" It defines a series of deterministic
rules based on inputs from the **Telemetric Endpoints**.

\"logic_tree\": {\
\"primary_input\": \"TE-MOISTURE-01\",\
\"secondary_input\": \"TE-FLOW-04\",\
\"rules\": \[\
{\
\"condition\": \"if moisture \< 28.0 AND humidity \> 15%\",\
\"action\": \"OPEN_VALVE_A\",\
\"duration_minutes\": 45,\
\"contingency\": \"IF flow_rate \< 5gpm THEN CLOSE_VALVE_A AND
ALERT_ERR_04\"\
},\
{\
\"condition\": \"if moisture \> 35.0\",\
\"action\": \"CLOSE_ALL_VALVES\",\
\"priority\": \"HIGH\"\
}\
\],\
\"safe_state\": \"ALL_VALVES_CLOSED\"\
}

## 5. Provenance & Audit Ledger (IRP Hooks)

To satisfy the **QCV (Quantifiable Conservation Validation)**, the Hub
must log every state change against these specific pointers.

\"audit_requirements\": {\
\"log_frequency_ms\": 60000,\
\"required_telemetry\": \[\"VWC\", \"EC\", \"FLOW\", \"BATT\"\],\
\"provenance_token\": \"IRP-TOKEN-9982-AX\"\
}

## 6. Serialization Standards

-   **Format:** Compressed JSON (GZIP) for transmission; parsed into
    > local memory-mapped structures on the Hub.

-   **Encryption:** AES-256-GCM using the hardware-specific Private Key
    > established during the **System Integration** phase.
